多条记录调用

<sgcms:getData model="" name="" id="" limit="" sql="" order="" where="" offset="" length="" mod="" key="">
循环内容
</sgcms:getData>
 
红色部分为必须项，各参数顺序不分前后
model：模块名称
name：属性表示模板赋值的变量名称,因此不可随意在模板文件中改变
id：表示当前的循环变量，可以随意指定，但确保不要和name属性冲突
limit：表示取得条数，也可以是范围 limit="5,10" 从第5条开始，取到第10条
sql：自行编写的SQL语句
order：排序。如： id DESC 或 id ASC 或id DESC,time ASC
where：搜索条件。如：categoryId=1
offset
length
 支持输出部分数据，例如输出其中的第5～15条记录
<sgcms:getData model="News" name="list" id="vo" offset="5" length='10'>
循环内容
</sgcms:getData>
mod：输出偶数记录
 <sgcms:getData model="News" name="list" id="vo" mod="2" >
<eq name="mod" value="1">{$vo.name}</eq>
</sgcms:getData>
Mod属性还用于控制一定记录的换行，例如：
<sgcms:getData model="News" name="list" id="vo" mod="5" >
<eq name="mod" value="4"><br/></eq>
</sgcms:getData>
key：输出循环变量
 

单条记录调用

<sgcms:getItem model="" name="" where="" sql="" />
 
红色部分为必须项
model：模块名称
name：属性表示模板赋值的变量名称,因此不可随意在模板文件中改变
where：搜索条件。如：id=1
sql：自行编写的SQL语句
最后必须以 />结尾
